# 一份可直接开跑的研究手册

这套代码已经把你要的目标拆成了四件事，并且每件事都能独立运行：

1. 训练 `ResNet-152`（120 epoch，ImageNet）。
2. 训练 `ResNet-152 + Stochastic Depth`（120 epoch，ImageNet）。
3. 对两个模型都做“从后往前删残差块”，画出 validation error 曲线。
4. 在可接受误差预算下，自动选择删层数，实现推理加速。

---

## 5. 单独运行命令（便于做消融）

### 5.1 只训练标准 ResNet-152

```bash
python train_imagenet.py \
  --data-root /path/to/imagenet \
  --output-dir outputs \
  --model resnet152 \
  --epochs 120 \
  --device cuda
```

### 5.2 只训练 Stochastic Depth 版本

```bash
python train_imagenet.py \
  --data-root /path/to/imagenet \
  --output-dir outputs \
  --model resnet152_sd \
  --sd-p-last 0.5 \
  --epochs 120 \
  --device cuda
```

### 5.3 删层曲线（以标准模型为例）

```bash
python explore_deletion.py \
  --data-root /path/to/imagenet \
  --model resnet152 \
  --checkpoint outputs/resnet152/best.pt \
  --device cuda
```

### 5.4 推理加速校准（以 SD 模型为例）

```bash
python speedup_inference.py \
  --data-root /path/to/imagenet \
  --model resnet152_sd \
  --sd-p-last 0.5 \
  --checkpoint outputs/resnet152_sd/best.pt \
  --error-budget 0.5 \
  --device cuda
```

## 6. 结果怎么看（研究导向）

先看 `deletion_curve.csv`：

- `deleted_blocks`: 从尾部删掉多少残差块
- `active_blocks`: 还保留多少块
- `top1`: 当前 Top-1
- `val_error`: 当前验证误差（`100 - top1`）

常见现象：

1. 前期删少量尾部块，误差增长较慢。
2. 超过某个拐点后，误差快速上升。
3. `resnet152_sd` 往往比普通 `resnet152` 对删层更鲁棒（拐点更靠右）。

这是因为：

- 残差结构天然有 identity 路径，部分尾部块是“细化器”而不是“必要主干”。
- Stochastic Depth 在训练时就随机短路块，模型被迫学会对层缺失更稳定的表征。
- 但删太多会丢失语义细节融合能力，误差会陡增。

## 7. 能不能用于推理加速？

可以，而且这套代码已经实现了一个可落地方案：

1. 先拿完整模型做基线误差和吞吐。
2. 逐步增加删层数。
3. 只要 `delta_error_vs_full <= 误差预算`，就继续删。
4. 输出满足预算下的最大删层配置（`recommended.txt`）。

你可以把 `recommended.txt` 里的 `active_blocks` 固化到部署配置里，得到稳定加速。

## 8. 关键实现细节（便于复核正确性）

- ResNet-152 深度配置是标准 `[3, 8, 36, 3]`。
- Stochastic Depth 使用线性生存率：
  `p_l = 1 - (l / L) * (1 - p_last)`，默认 `p_last=0.5`。
- 训练阶段：每个残差块按 Bernoulli 生存率随机保留或短路。
- 测试阶段：按期望缩放残差分支（乘以 `p_l`）。
- 删层方式：统一通过 `set_active_block_count()` 从网络尾部裁剪块。

## 9. 常见问题

### Q1: 显存不够怎么办？

- 降低 `--batch-size`（如 256 -> 128 -> 64）。
- 同时可降低 `--val-batch-size`。

### Q2: 想更快得到趋势，不跑满 120 epoch 可以吗？

- 可以先小规模试跑（例如 10-30 epoch）验证脚本流程。
- 真正结论建议跑满 120 epoch后再比较两条删层曲线。

### Q3: 如何调整“精度-速度”偏好？

- 改 `speedup_inference.py` 的 `--error-budget`。
- 预算越宽松，允许删更多层，速度提升更大。
